# Write RFCs Code

This package models RFC writing as explicit problem framing, alternatives, risks, migration planning, and lifecycle status instead of unstructured design prose.

## Coverage

- RFC section completeness review
- alternatives and tradeoff validation
- migration and rollback planning
- status and decision tracking
- implementation linkage expectations
- maintainability-oriented RFC stewardship

## Notes

- this is an enterprise-style RFC package with local validation support
- the goal is to make design proposals reviewable before implementation starts
- files are intentionally structured to separate document shape, review policy, and lifecycle status
