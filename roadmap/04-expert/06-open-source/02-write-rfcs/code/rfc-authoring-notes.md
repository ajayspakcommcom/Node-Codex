# RFC Authoring Notes

## Review Sequence

1. State the problem and decision scope clearly.
2. Record non-goals and alternatives.
3. Capture risks, migration, and rollback steps.
4. Review stakeholder impact and unresolved questions.
5. Link the RFC to implementation status after approval.

## Rules

- RFCs must explain why the change exists, not only what it does
- alternatives and tradeoffs are required for meaningful review
- migration and rollback plans are part of the proposal, not afterthoughts
- RFC status should stay aligned with implementation reality
