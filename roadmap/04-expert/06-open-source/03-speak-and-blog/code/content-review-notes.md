# Content Review Notes

## Review Sequence

1. Define the audience and practical takeaway.
2. Check that the material is supported by real engineering evidence.
3. Remove or rewrite confidential details while preserving the lesson.
4. Validate structure so the talk or article stays useful to practitioners.
5. Publish only after technical and confidentiality review pass.

## Rules

- public content needs a clear technical lesson, not just a story
- claims about scale or impact must be evidence-backed
- sensitive implementation details must be removed or generalized
- public content should be reviewed with the same rigor as important documentation
