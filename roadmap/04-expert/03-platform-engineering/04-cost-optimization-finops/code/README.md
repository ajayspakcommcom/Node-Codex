# Cost Optimization (FinOps) Code

This package models cost optimization as explicit ownership, budget policy, anomaly review, and guardrail decisions instead of after-the-fact cloud billing analysis.

## Coverage

- service and team cost attribution
- budget policy and budget breach detection
- anomaly review for unexpected cost spikes
- platform guardrails for wasteful defaults
- unit-cost awareness for service operations
- maintainability-oriented cost governance

## Notes

- this is an enterprise-style FinOps package with local validation support
- the goal is to keep cost review actionable for engineering teams rather than finance-only reporting
- files are intentionally structured to separate attribution, budgets, anomalies, and guardrails
