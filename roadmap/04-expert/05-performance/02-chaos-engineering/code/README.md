# Chaos Engineering Code

This package models chaos engineering as explicit hypotheses, steady-state checks, blast-radius control, and abort logic rather than ad hoc failure injection.

## Coverage

- hypothesis definition
- steady-state validation
- blast-radius control
- abort-condition checks
- resilience-gap review
- maintainability-oriented experiment ownership

## Notes

- this is an enterprise-style chaos-engineering package with local validation support
- the goal is to keep experiments reviewable, bounded, and actionable
- files are intentionally structured to separate hypotheses, safeguards, and result interpretation
