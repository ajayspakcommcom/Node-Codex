# Chaos Review Notes

## Review Sequence

1. Define the failure hypothesis and expected steady state.
2. Limit blast radius to a safe scope.
3. Define the signals to watch during the experiment.
4. Set explicit abort conditions before injection starts.
5. Record resilience gaps and follow-up actions after the run.

## Rules

- every experiment needs a hypothesis, not just a failure type
- blast radius must be explicit and bounded
- abort logic must be pre-defined and testable
- follow-up work is part of the experiment outcome, not optional cleanup
