# Compliance (SOC 2, GDPR) Code

This package models compliance as explicit control mapping, evidence generation, retention rules, and deletion workflows rather than post-build audit paperwork.

## Coverage

- control ownership and mapping
- automated evidence generation
- data classification and retention policy
- GDPR-style deletion workflow validation
- compensating control and review logic
- maintainability-oriented compliance boundaries

## Notes

- this is an enterprise-style compliance package with local validation support
- the goal is to keep compliance tied to real engineering controls and system behavior
- files are intentionally structured to separate control mapping, evidence, retention, and deletion workflows
