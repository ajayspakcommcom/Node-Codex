# Compliance Control Notes

## Review Sequence

1. Map each important requirement to a concrete technical control.
2. Assign owners for control operation and evidence generation.
3. Validate retention and deletion rules against data classification.
4. Review exceptions and compensating controls explicitly.
5. Keep evidence generation tied to normal system behavior where possible.

## Rules

- controls without owners are not reliable controls
- evidence should be generated by systems, not assembled manually after the fact
- regulated data needs explicit retention and deletion treatment
- exceptions require documented compensating controls and review dates
